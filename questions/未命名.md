# 🧠 编译语言 vs 解释/VM语言在虚拟内存上的本质区别

> [!summary] 核心结论
> - **编译型语言（C/Rust）**：直面物理系统。**代码即代码**（直接映射到只读代码段），**栈即物理栈**（由 CPU 寄存器直接驱动），内存由 OS 和代码逻辑直接控制。
> - **解释/VM型（Python/Java）**：套娃系统。**代码即数据**（你的代码被当作数据读入堆中），由只读代码段里的“解释器程序”去读取并翻译执行；**栈是模拟栈**（经常在堆中分配对象来模拟函数调用栈）。

### 📊 维度深度对比表

| 对比维度 | 编译型语言 (Compiled) | 解释/VM型语言 (Interpreted / VM) |
| :--- | :--- | :--- |
| **代码段 (`.text`) 存放内容**| 本程序的原生机器指令（Native Code） | 解释器/虚拟机（如 `python` / `java`）自身的机器指令 |
| **用户代码在内存中的形态** | 机器码（只读，直接被 CPU 执行） | 字节码或 AST（作为数据存放在**堆**中，被解释器读取） |
| **栈（Stack）的物理本质** | 操作系统原生分配的栈（高效、硬件级支持） | 往往在**堆（Heap）**中通过对象模拟分配虚拟栈帧 |
| **动态内存管理分配者** | 操作系统内核（通过 `brk`/`mmap` 系统调用）| 虚拟机自身的内存管理器（虚拟机一次性向 OS 批发出大片内存，自己再在堆里细分） |
| **JIT 编译器的内存特权** | 通常不需要（静态编译完毕） | 需要向 OS 申请 `W^X`（可写且可执行）的特殊虚拟内存页 |
